### 亮点 · 值得在面试主打（复核确认为真实优势）

- **架构判断力（最该主打）**：`scoring.ts` 全程确定性、`breakdown` 摊给用户；`useExplain`（`hooks.ts:140-222`）有 **550ms 防抖 + 模块级缓存 + 静默降级到规则理由**，UI 还区分 deepseek/template 来源。天气永远来自真实 API，LLM 不碰决策——"LLM 是嘴和耳朵、规则是大脑、全程不白屏"这条**代码里扎实兑现**。
- **香水专业度**：中文映射规范（Immortelle=蜡菊、Opoponax=甜没药、Styrax=苏合香、Ambroxan=降龙涎香醚、Tonka=零陵香豆、Oud=沉香/乌木），无外行硬伤；留香只给区间、社交距离绑 sillage 四档、**无"揉搓破坏前调"伪科学**（全库无 rub 逻辑）；天气家族用互斥 feel 分支，**无"既算 heavy 又算 warm"的双重计数**。
- **数据管线正确**：拉普拉斯平滑（α=5，四季和恒为 1）、daypart 除零保护、sillageTier 分档、Top-N 选取（`people≥50` + `popularity.magnitude`）均正确、边界都处理了。
- **前端信息架构**：三 Tab 扁平、今日卡"一屏结论 + 语义化 `<details>` 展开"真的砍掉了二级页；动效克制、尊重 `prefers-reduced-motion`、**切主题瞬间双 rAF 禁过渡防闪**——"高级感来自细节"的实证。
- **降级工程**：三条 API 几乎所有失败都返回 200+`{error}`，前端统一判失败不冒泡白屏；`reqId` 竞态守卫防过期响应覆盖。
- **边界稳健**：小库/同品牌备选去重有 fallback while 自愈，不产空备选、不越界。
